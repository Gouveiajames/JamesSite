Mamma Donna's Daycare a React.js app

California State University Sacramento requires all Computer Science students to create a final project. This capstone project is intended to give students an opportunity to solve a real-world problem using real world techniques such as Scrum and the Agile methodology. My team consisted of a team of 8 students. We named our team the NULL Pointers! We found a client, Donna, who needed a software solution to run her small day care. The team took up the project! Over the course of two semesters, we went from wireframes and diagrams to a working web app! Please see below for more specific information about this project.

Problem:

Donna needs an all-in-one day care business management solution for a home based day care

Requirements:

  • Child Record Keeping
    • Attendance Log
    • Student Records
    • Breathing Check Log
    • Diaper Check Log
  • Accounting
    • Basic Bookeeping - Income/Expense
    • Automatic Invoicing
    • Accept Pay Pal payment
  • Scheduling Tool
    • Accept/Deny Requests
    • Maintain a schedule of planned student attendance
  • Client Communication
    • Ability to send/receive messages from clients
    • Ability to send daily lesson plans to clients

Solution:

The team explored many options from a cell phone app to a web solution. We settled on a web app for several reasons. A phone app has a cost associated with it to be hosted on the app store and every penny counts for a new business. The other major consideration was incompatibility between the framework we wanted to use, and the language needed to build an app that works on Android or IOS. With these considerations in mind, we decided to build a web app using Firebase. Firebase has built in authentication and an easy interface to the backend database. In addition, for the application we intended the service would be free to host and run which is mission critical for our small business application.

Architecture:

Each of the core functions such as messaging, or logs have their own separate pages of code. The pages are connected to either the admin or client portal. We did our best to not repeat code and modularize the app as much as possible. We made use of helper functions such as hooks to help prevent redundant code.
